What  changes  compared  to  Model  1  
In  Model  1,  the  backend  may  directly  trigger  background  tasks.  
In  Model  2,  the  backend  publishes  events.  
Example:  
User  uploads  report  →  FastAPI  saves  metadata  
--- PAGE 7 ---
→  FastAPI  publishes  ReportUploaded  event  →  OCR  worker  consumes  event  →  OCR  worker  extracts  text  →  Embedding  worker  chunks  +  embeds  →  Summary  worker  creates  report  summary  →  Notification  worker  informs  user  
This  is  much  more  reliable.  
Event  examples  ReportUploaded  OCRCompleted  EmbeddingCompleted  ReminderDue  NotificationRequested  PaymentWebhookReceived  DailySummaryRequested  GoalProgressUpdated  
Why  this  is  better  for  expansion  
Because  future  features  can  plug  into  events.  
Example:  
Later,  when  you  add  weekly  insights:  
GoalProgressUpdated  →  Insights  worker  listens  →  Generates  weekly  gap  analysis  
Later,  when  you  add  family  accounts:  
CriticalReminderMissed  →  FamilyAlert  worker  listens  →  Sends  alert  to  family  member  
You  do  not  need  to  rewrite  the  whole  system.

7.6  Request  Flow  —  AI  Chat  
Step  1:  User  asks  a  question.  
Step  2:  Backend  verifies  authentication.  
Step  3:  Backend  retrieves  user  profile  summary.  
Step  4:  Backend  retrieves  relevant  embeddings  from  Vertex  AI  Vector  Search.  
Step  5:  Prompt  builder  creates  a  safe  prompt.  
Step  6:  Vertex  AI  Gemini  generates  answer.

Step  8:  Response  is  stored  in  MongoDB.  
Step  9:  Response  is  returned  to  user.

Step  3:  Report  metadata  is  stored  in  MongoDB.  
Step  4:  Document  AI  extracts  text.  
--- PAGE 43 ---

Step  8:  Gemini  generates  summary.  
Step  9:  User  is  notified  when  report  is  ready.

12.  Glossary  
Modular  Monolith  
A  single  backend  application  that  is  internally  divided  into  clean  modules.  It  is  easier  to  build  
than
 
microservices
 
but
 
still
 
maintainable.
 
Microservices  
--- PAGE 51 ---
An  architecture  where  each  business  domain  runs  as  an  independent  service.  It  improves  
scaling
 
but
 
increases
 
complexity.
 
High  Availability  
Designing  the  system  so  it  continues  working  even  if  one  server,  container,  or  Availability  Zone  
fails.
 
Availability  Zone  
A  physically  separate  data  center  area  inside  a  cloud  region.  Running  services  across  multiple  
AZs
 
improves
 
reliability.
 
Load  Balancer  
A  service  that  distributes  traffic  across  multiple  backend  instances.  
AWS  WAF  
AWS  Web  Application  Firewall.  It  protects  web  applications  from  common  attacks.  
Redis  
An  in-memory  data  store  used  for  caching,  rate  limiting,  locks,  and  short-lived  temporary  data.  
TTL  
Time  to  Live.  It  defines  how  long  cached  data  stays  before  being  automatically  deleted.  
MongoDB  Atlas  
A  managed  MongoDB  database  service.  
AWS  S3  
Object  storage  service  used  for  storing  files  such  as  PDFs,  images,  and  documents.  
RAG  
--- PAGE 52 ---
Retrieval-Augmented  Generation.  It  retrieves  relevant  information  from  documents  before  
sending
 
context
 
to
 
an
 
LLM.
 
Embeddings  
Numerical  representations  of  text  used  for  similarity  search.  
Vector  Search  
A  search  method  that  finds  semantically  similar  text  chunks  using  embeddings.  
OCR  
Optical  Character  Recognition.  It  extracts  text  from  images  or  PDFs.  
AI  Orchestrator  
The  backend  component  that  decides  how  to  build  prompts,  retrieve  context,  call  LLMs,  apply  
guardrails,
 
and
 
return
 
responses.
 
EventBridge  
AWS  event  bus  service  used  for  event-driven  architecture.  
SQS  
AWS  Simple  Queue  Service.  It  is  used  for  reliable  background  job  queues.  
Dead-Letter  Queue  
A  queue  where  failed  messages  go  after  retry  attempts  are  exhausted.  
Outbox  Pattern  
A  reliability  pattern  where  events  are  first  stored  in  the  database  before  being  published,  
preventing
 
lost
 
events.
 
RBAC  
--- PAGE 53 ---
Role-Based  Access  Control.  It  controls  what  users  can  access  based  on  their  role.  
Audit  Log  
A  record  of  important  actions  such  as  report  access,  admin  actions,  login  events,  and  data  
deletion.
 
PHI  
Protected  Health  Information.  Sensitive  health-related  user  data  that  must  be  handled  carefully.  
Disaster  Recovery  
A  plan  to  restore  the  system  after  major  failure,  data  loss,  or  cloud  outage.
